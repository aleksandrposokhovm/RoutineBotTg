# RoutineBot — Персональный ИИ-ассистент расписания, планов и рациона

**RoutineBot** — это интеллектуальный ассистент в Telegram и встроенное веб-приложение (Telegram Mini App), которое помогает управлять повседневными задачами, расписанием встреч и отслеживать рацион питания (КБЖУ). Проект интегрирован с API **Gemini** (модель `gemini-2.5-flash`) для распознавания естественного языка (как текста, так и голосовых сообщений) и автоматического анализа фотографий еды.

## Архитектура и стек технологий

Проект разделен на две основные части: монолитный бэкенд и фронтенд для Telegram Mini App.

### Бэкенд (`/backend`)
*   **Среда выполнения и язык:** Node.js, TypeScript.
*   **Веб-сервер:** Express (API-эндпоинты для фронтенда).
*   **Telegram-бот:** Telegraf (интеграция с Telegram Bot API, обработка текстовых/голосовых сообщений и изображений).
*   **База данных:** SQLite через Prisma ORM.
*   **Интеграция с ИИ:** `@google/genai` (Gemini 2.5 Flash).
    *   *Обработка текста и голоса:* Извлечение намерений (Intent detection) для создания, редактирования, завершения и удаления задач, расписания, приемов пищи.
    *   *Компьютерное зрение:* Анализ фотографий еды, определение названия блюда, примерного веса порции и расчет КБЖУ.
    *   *Генерация ответов:* Формирование красивых и структурированных сводок о расписании и питании на естественном языке с использованием Markdown.
    *   *Расчет КБЖУ:* Автоматический расчет суточной нормы по формуле Миффлина-Сан Жеора.
*   **Планировщик:** Система отправки напоминаний о важных событиях (`scheduler.ts`).

### Фронтенд (`/frontend`)
*   **Технологии:** React (TypeScript), Vite в качестве сборщика.
*   **Стилизация:** Современный и отзывчивый интерфейс на чистом CSS (`index.css`) с поддержкой темных тем и микроанимаций.
*   **Интеграция:** Telegram WebApp SDK для бесшовной работы внутри Telegram-интерфейса.
*   **Основные разделы:**
    1.  **Расписание (📅):** Управление событиями с жесткой привязкой ко времени в часовом поясе пользователя.
    2.  **План (📋):** Список To-Do задач на конкретный день.
    3.  **Рацион (🍽):** Управление приемами пищи, отслеживание суточной калорийности и баланса макронутриентов (БЖУ) относительно нормы.

## Структура базы данных (Prisma Schema)

*   `User`: Хранит информацию о пользователе Telegram (id, имя, часовой пояс).
*   `ScheduleEvent`: События с точным временем начала и конца (время хранится в UTC, отображается в часовом поясе пользователя), статус напоминаний.
*   `Plan`: Список To-Do задач на конкретный день с отметкой о выполнении.
*   `DietEntry`: Записи о съеденном (название, тип приема пищи, калории, белки, жиры, углеводы, вес порции, id фотографии).
*   `NutritionProfile`: Персональные параметры пользователя (рост, вес, возраст, уровень активности, цель) и рассчитанные суточные лимиты КБЖУ.

## Установка и запуск

### 1. Настройка окружения
В папке `backend/` скопируйте `.env.example` в `.env` и заполните переменные:
```env
PORT=3001
DATABASE_URL="file:./dev.db"
TELEGRAM_BOT_TOKEN="ваш_токен_телеграм_бота"
# Google Gemini API Keys (можно указать несколько через запятую для авторотации: KEY1,KEY2,KEY3...)
GEMINI_API_KEY="первый_ключ,второй_ключ"
FRONTEND_URL="http://localhost:5173"
JWT_SECRET="ваш_секретный_ключ_для_jwt"
```

### 2. Подготовка базы данных
Перейдите в папку `backend` и выполните команды для генерации клиента Prisma и создания таблиц:
```bash
cd backend
npm install
npm run db:generate
npm run db:push
```

### 3. Запуск бэкенда (Development)
```bash
npm run dev
```
Сервер запустится на порту `3001` (или другом, указанном в `.env`). Также автоматически стартует Telegram-бот и планировщик напоминаний.

### 4. Запуск фронтенда (Development)
Перейдите в папку `frontend`, установите зависимости и запустите локальный сервер Vite:
```bash
cd ../frontend
npm install
npm run dev
```
Фронтенд будет доступен по адресу `http://localhost:5173`.

## 🚀 Развертывание на удаленный сервер (Production)

В продакшене бэкенд настроен на **автоматическую раздачу собранного фронтенда** (если папка `frontend/dist` скомпилирована), поэтому вам не требуется настраивать отдельный веб-сервер для статики. Все работает на едином порту бэкенда (по умолчанию `3001`).

### Вариант 1: С использованием PM2 (Рекомендуется для VPS)

PM2 — менеджер процессов для Node.js, который будет держать бота запущенным 24/7 и автоматически перезапустит его при сбое или перезапуске сервера.

1. **Установите PM2 глобально на сервере (если не установлен):**
   ```bash
   sudo npm install -y -g pm2
   ```

2. **Подготовьте конфигурацию:**
   Склонируйте проект, скопируйте `backend/.env.example` в `backend/.env` и заполните переменные окружения (BOT_TOKEN, GEMINI_API_KEY, ALLOWED_TG_IDS и т.д.).
   *Примечание:* Установите `PORT=3001` и `FRONTEND_URL` укажите адрес вашего сервера/бота (например, `https://yourdomain.com`).

3. **Выполните деплой:**
   Мы подготовили скрипт `deploy.sh` для автоматизации всего процесса сборки и запуска. Выполните его в корне проекта:
   ```bash
   ./deploy.sh
   ```
   Этот скрипт выполнит `git pull`, установит зависимости фронтенда и бэкенда, обновит базу данных Prisma SQLite, скомпилирует TypeScript файлы и запустит/перезагрузит процесс в PM2.

4. **Полезные команды PM2:**
   - Посмотреть статус приложения: `pm2 status`
   - Логи в реальном времени: `pm2 logs routinebot`
   - Перезапустить приложение вручную: `pm2 restart routinebot`
   - Настроить автозапуск PM2 при перезагрузке VPS: `pm2 startup` и затем `pm2 save`

---

### Вариант 2: С использованием Docker и Docker Compose

Если вы предпочитаете контейнеризацию, проект полностью готов к работе в Docker. База данных SQLite будет сохраняться на хосте в папке `backend/prisma/` благодаря примонтированному volume.

1. **Подготовьте окружение:**
   Создайте и заполните файл `backend/.env` (Docker Compose использует его автоматически).

2. **Запустите контейнеры:**
   В корне проекта выполните команду:
   ```bash
   docker compose up -d --build
   ```
   Эта команда выполнит многоэтапную сборку (соберет фронтенд и бэкенд в Docker) и запустит контейнер в фоновом режиме.

3. **Обновление приложения:**
   Для обновления до последней версии кода просто выполните:
   ```bash
   git pull && docker compose up -d --build
   ```

4. **Просмотр логов:**
   ```bash
   docker compose logs -f
   ```
